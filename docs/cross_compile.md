# Raspberry Pi Cross-Compilation Guide

This guide explains how to cross-compile, deploy, and run the project on Raspberry Pi from an Ubuntu host.

The workflow uses:

- Ubuntu host for development and cross-compilation
- Raspberry Pi Zero 2W as the runtime target
- ARM64 (aarch64) sysroot synchronized from the Pi
- vendored ARM64 OpenCV build
- vendored ARM64 TensorFlow Lite build
- staged installation and deployment

---

### Configure SSH Access

Generate an SSH key on the Ubuntu host:

```bash
$ ssh-keygen -t ed25519
```
      
Copy the key to the Raspberry Pi :

```bash
$ ssh-copy-id <pi_user>@<pi_host>.local
```

Create an SSH shortcut: :

```bash
$ mkdir -p ~/.ssh
$ nano ~/.ssh/config
```

Example config:

```
Host zero2w
    HostName zero2w.local
    User pi
```

Test:

```bash
ssh zero2w
```

### Enable Passwordless Sudo on Pi

This enables setup scripts to run remotely.

Run on Pi:

```bash
sudo visudo
```

Add this line at the bottom of the opened file:  

```
pi ALL=(ALL) NOPASSWD: ALL
```

Only do this on trusted development devices.

### Full Bootstrap Workflow for Raspberry Pi

Make all scripts executable:

```bash
chmod +x ./scripts/*
```

Use `bootstrap_pi.sh` for the first full setup.
Run from the Ubuntu host:

```bash
./scripts/bootstrap_pi.sh <PI_HOST> [target_dir] [build_type]
```

Example:

```bash
./scripts/bootstrap_pi.sh zero2w motion-tracker Release
```

This script may take some time running, as it:

- Configures Raspberry Pi dependencies
- Synchronizes the ARM64 sysroot
- Builds ARM64 OpenCV
- Builds ARM64 TensorFlow Lite
- Cross-compiles the project
- Deploys binaries and runtime libraries


Equivalent manual steps:

```bash
./scripts/setup_host.sh
scp -p ./scripts/setup_pi.sh <PI_HOST>:/home/pi/
ssh zero2w "/home/pi/setup_pi.sh && rm /home/pi/setup_pi.sh"
./scripts/sync_sysroot.sh <PI_HOST>
./scripts/build_opencv.sh
./scripts/build_tflite.sh
./scripts/build_pi.sh [target_dir] [build_type]
./scripts/deploy_pi_full.sh <PI_HOST> [target_dir]
```

### Sysroot

The sysroot is a local copy of required Raspberry Pi headers and libraries.

It is stored in `./sysroot/` and generated by:

```bash
./scripts/sync_sysroot.sh <PI_HOST>
```

The cross-compiler uses it through:

```text
-DCMAKE_SYSROOT=./sysroot
```

Sysroot allows the Ubuntu host to compile against the same libraries available on the Pi.

### OpenCV Cross-compilation

OpenCV is cross-compiled on the Ubuntu host for Raspberry Pi ARM64:

```bash
./scripts/build_opencv.sh
```

Installed into:

```text
third_party/opencv/aarch64-linux-gnu
```

Enabled modules:
- core
- imgproc
- imgcodecs
- videoio
- highgui  
- video

Enabled features:

- GStreamer
- V4L2
- FFMPEG

OpenCV libraries are built with:

```text
RUNPATH=$ORIGIN
```

allowing deployed OpenCV shared libraries to resolve their dependencies from the local deployment directory.

### TensorFlow Lite
TensorFlow Lite is built from source using Bazel:

```bash
./scripts/build_tflite.sh
```

Installed into:

```text
third_party/tflite/aarch64-linux-gnu
```

The project uses:

- TensorFlow Lite
- XNNPACK delegate
- ARM NEON optimizations

### Install Layout 

Cross-compilation produces a staged installation:

```
./install/<target_dir>/usr/
├── bin/
│   └── motion_tracker
├── lib/
│   └── runtime libraries
└── share/
    └── motion-tracker/
        ├── config.yaml
        └── models/
```

Deployment copies the contents of `usr/` to:

```text
/home/pi/<target_dir>/
```

### Normal Rebuild Workflow

After the first bootstrap, use this when only project source code changed:
    
```bash
./scripts/build_pi.sh <target_dir> [build_type]
./scripts/deploy_pi_binary.sh <PI_HOST> <target_dir>
```
Use bootstrap again only when:

- Pi packages changed
- sysroot is outdated

### Run on Raspberry Pi

```bash
ssh zero2w
cd /home/pi/motion-tracker
./run.sh
```

### When to Use Each Script

| Script                  | Purpose                                      |
|-------------------------|----------------------------------------------|
| setup_host.sh           | Install host build tools and dependencies    |
| bootstrap_pi.sh         | Complete first-time setup workflow           |
| setup_pi.sh             | Install Raspberry Pi dependencies            |
| sync_sysroot.sh         | Synchronize Raspberry Pi sysroot             |
| build_opencv.sh         | Build ARM64 OpenCV                           |
| build_tflite.sh         | Build ARM64 TensorFlow Lite                  |
| build_pi.sh             | Cross-compile the application                |
| deploy_pi_full.sh       | Deploy full runtime environment              |
| deploy_pi_binary.sh     | Deploy only the application binary           |
